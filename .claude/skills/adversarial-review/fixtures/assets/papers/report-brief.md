# Report brief — efficient-ML literature summary

Write a short **literature review** as a brand-new `.docx` from the three papers in
this folder (`paper-1.md`, `paper-2.md`, `paper-3.md`). There is no template — you
are authoring from scratch.

## Required structure

1. A **title** ("Techniques for Efficient Model Deployment: A Brief Review") and a
   one-line byline/date.
2. A short **Introduction** paragraph framing the theme (running capable models
   under tight compute/memory budgets).
3. One **section per paper** (use real heading styles), each with:
   - a heading naming the technique,
   - 2–3 sentences summarizing the approach and headline result,
   - an **in-text citation** to the authors and year (e.g., "(Nguyen et al., 2025)").
4. A **"Comparison"** section that embeds the chart **`figure-1.png`** (provided in
   this folder) as a figure, with a caption ("Figure 1. Accuracy retained vs.
   baseline across the three methods.").
5. A **References** section listing all three papers in a consistent format.

## What "done" looks like

A multi-section report with proper headings, the three summaries with citations, the
embedded figure rendering correctly, and a references list. Save it as
**`report.docx`** in this scenario folder.
